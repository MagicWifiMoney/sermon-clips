## 1. TikTok Slideshow/Carousel Format Best Practices

### Technical Specifications

#### Image Requirements
- **Recommended Size:** 1080 x 1920 pixels
- **Aspect Ratio:** 9:16 (vertical) preferred, 1:1 (square) and 4:5 also supported
- **File Formats:** JPG, PNG, JPEG, WebP
- **File Size Limits:**
  - Maximum per image: 10-20 MB
  - Total carousel limit: 500 MB
- **Image Count:** 
  - Minimum: 2 images (carousel ads), 4 images (organic posts)
  - Maximum: 35 images (organic), 10 images (carousel ads)
  - **Optimal for engagement:** 5-10 slides

#### Photo Mode vs. Video Mode
- **Photo Mode:** Creates true carousel with manual swiping
- **Video Mode:** Auto-playing slideshow (different algorithm treatment)
- Note: Photo Mode is specifically for images only - cannot mix video and images

### Content Best Practices

#### Visual Design
- **Consistency is critical:** Use same color scheme, fonts, and filters across all slides
- **High-resolution only:** Blurry or pixelated images kill engagement
- **Vertical format preferred:** 9:16 fills the screen without letterboxing
- **First slide = hook:** Must be visually captivating and indicate value
- **Design for mobile:** Always preview on phone screen
- **Text overlays:** Keep concise, use readable fonts with sufficient contrast
- **White space matters:** Don't overcrowd slides

#### Storytelling Structure
- **Slide 1:** Strong hook (question, statistic, compelling visual)
- **Slides 2-N:** Logical progression that encourages swiping
- **Final slide:** Summary, CTA, or satisfying conclusion
- Think of it as a mini-storyboard or flip book

#### Text Overlay Rules
- Keep text minimal and scannable
- Use large, bold fonts
- Ensure readability at a glance
- Avoid walls of text (users swipe quickly)
- Text can help algorithm understand content context

### Optimal Carousel Length

**Data-backed recommendations:**
- **3-10 slides:** Sweet spot for most content
- **Too few (<3):** Limited storytelling potential
- **Too many (>15):** Risk of user drop-off
- **Quality > Quantity:** Each slide must add value

**By content type:**
- Tutorials/How-tos: 5-8 slides
- Product showcases: 3-6 slides  
- Before/after transformations: 3-5 slides
- Listicles: 5-10 slides
- Stories/narratives: 6-12 slides

## 2. TikTok Algorithm Behavior: Slideshows vs. Video (2025-2026)

### Performance Comparison Data

**Carousel posts vs. Video posts (2025 data):**

| Metric | Carousels | Videos | Difference |
|--------|-----------|--------|------------|
| Reach | +3% higher | Baseline | Carousels win slightly |
| Total Engagement | +81% higher | Baseline | **Massive carousel advantage** |
| Likes | +82% more | Baseline | **Massive carousel advantage** |
| Comments | 2.9x more | Baseline | **Massive carousel advantage** |
| Shares | 2.6x more | Baseline | **Massive carousel advantage** |
| Views per post | Similar/higher | Baseline | Roughly equal |

**Key Insight:** TikTok is actively promoting photo carousels. The platform is pushing creators to share more slideshows, promising visibility boosts and FYP placement.

### Algorithm Signals for Carousels

The TikTok carousel algorithm analyzes unique engagement signals:

#### 1. Swipe-Through Behavior
- **Swipe-Through Rate (STR):** Did users swipe to slide 2, 3, etc.?
- **Completion Rate:** How many users viewed ALL slides?
- **Dwell Time per Slide:** How long users spend on each image
- **Reverse Swipes:** Strong positive signal when users swipe back to re-view

#### 2. Initial Distribution Test
- First few hours are **critical**
- Algorithm shows carousel to small test group (followers + similar users)
- Measures **velocity of engagement** (how fast likes/comments come in)
- High early performance = wider distribution to FYP

#### 3. Classic Engagement Metrics (Still Matter)
- Likes
- Comments (sentiment also analyzed)
- Shares (powerful signal for reach expansion)
- Saves/Favorites (indicates valuable, evergreen content)

### Why Carousels Outperform Videos

**Platform strategic push:**
- TikTok launching "TikTok Notes" app (photo-focused Instagram competitor)
- Photo posts = lower barrier to entry for creators
- TikTok wants to be "all things to all people"

**User behavior shift:**
- Audiences experiencing video fatigue
- Carousels allow self-paced consumption
- Higher dwell time = better algorithm signals
- Lower production barrier = more consistent posting

**Algorithm advantages:**
- Each swipe = engagement signal
- Longer time on post (multiple slides viewed)
- More opportunities for text/context analysis
- Better for educational/informational content

### Algorithm Evolution (2025)

**Key changes:**
- **Slide-specific engagement tracking:** Algorithm now analyzes each individual slide's performance
- **Sophisticated image analysis:** AI "reads" image content, text overlays, visual quality
- **Niche-specific distribution:** Smarter at matching carousels to highly relevant audiences
- **E-commerce integration:** Algorithm optimized for product discovery via carousels
- **Cross-format learning:** Insights from video performance inform carousel distribution

### Algorithm Optimization Tips

1. **Post slightly before peak times:** Give algorithm time to test before audience arrives
2. **Encourage swipes explicitly:** Use CTAs like "Swipe to see..." or "Keep swiping for..."
3. **Trending sounds matter:** Even for photo posts, add trending audio
4. **Strategic hashtags:** 5-8 relevant hashtags (niche + broad mix)
5. **Caption length:** 200+ characters recommended for searchability
6. **Consistency wins:** Regular posting schedule signals account authority

## 4. TikTok Content API & Programmatic Posting

### Official TikTok Content Posting API

**Endpoint:** `POST /v2/post/publish/content/init/`

**Scope required:** `video.publish` or `video.upload`

**Rate limits:** 6 requests per minute per user access token

### Photo/Carousel Posting via API

**Supported parameters:**

```json
{
  "media_type": "PHOTO",
  "post_mode": "DIRECT_POST" or "MEDIA_UPLOAD",
  "post_info": {
    "title": "string (90 chars max for photos)",
    "description": "string (4000 chars max)",
    "privacy_level": "PUBLIC_TO_EVERYONE|MUTUAL_FOLLOW_FRIENDS|FOLLOWER_OF_CREATOR|SELF_ONLY",
    "disable_comment": boolean,
    "auto_add_music": boolean,
    "brand_content_toggle": boolean,
    "brand_organic_toggle": boolean
  },
  "source_info": {
    "source": "PULL_FROM_URL",
    "photo_images": ["array of 2-35 URLs"],
    "photo_cover_index": 0
  }
}
```

### Key API Capabilities & Limitations

**âœ… Supported:**
- Direct posting to TikTok (with `video.publish` scope)
- Upload for user editing (with `video.upload` scope)
- Up to 35 images per carousel
- Auto-add music (direct post only, not drafts)
- Privacy controls
- Comment settings
- Image URLs must be publicly accessible and verified

**âŒ Limitations:**
- Cannot add trending audio via API (must be done in-app)
- Auto-add music only works for scheduled posts, NOT drafts
- Duet/Stitch controls do NOT apply to carousels (video-only features)
- Title max 90 characters (vs. 4000 for description)
- Must verify URL ownership/domain for PULL_FROM_URL method
- Daily posting caps exist (spam prevention)
- Unaudited clients can only post to private accounts
- Photo-only format (cannot mix images and video in carousel)

**Quota limits:**
- Daily post cap per user (spam prevention)
- Daily active user cap per API client
- Upload quota: Max 5 pending shares per 24 hours

**Error handling:**
- 400: Invalid parameters
- 403: Spam risk, banned user, quota exceeded, unverified URL
- 429: Rate limit exceeded
- 401: Invalid/expired access token

### URL Transfer Requirements

If using `PULL_FROM_URL` method:
- Must verify domain/URL prefix ownership with TikTok
- URLs must be publicly accessible
- Failure to verify = `url_ownership_unverified` error

### Music/Audio Strategy via API

**Critical limitation:** Trending audio **CANNOT** be added directly via API.

**Workarounds:**
1. **Auto-add music feature** (API parameter):
   - `"auto_add_music": true` in API request
   - TikTok automatically adds recommended music
   - Only works for DIRECT_POST mode, not MEDIA_UPLOAD
   - User can change music later in TikTok app

2. **Notification-based posting** (Buffer, Later):
   - Schedule content via third-party tool
   - Receive push notification when it's time to post
   - Tool opens video in TikTok app
   - Manually add trending sound before publishing
   - Saves time while allowing trending audio

3. **Upload mode + manual completion:**
   - Use API to upload carousel with `"post_mode": "MEDIA_UPLOAD"`
   - User receives inbox notification in TikTok
   - Complete post in TikTok editing flow (add trending sound)

**Recommendation for trending audio:**
- Use notification-based posting (Buffer/Later) for maximum flexibility
- For automated workflows, rely on auto-add music feature
- Monitor trending sounds separately and plan content around them

### API vs. Manual Trade-offs

| Method | Pros | Cons |
|--------|------|------|
| **Direct API Posting** | Fully automated, scalable, consistent scheduling | Cannot add trending audio, limited editing |
| **Notification Posting** | Add trending audio manually, retain control | Requires manual action, less automated |
| **Upload + Complete** | Flexible editing, user control | Two-step process, inbox notification required |

**Practical reality:**
- Most users don't read long captions
- Short attention spans dominate the platform
- First 1-2 lines must hook immediately
